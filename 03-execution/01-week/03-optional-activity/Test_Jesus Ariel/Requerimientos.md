
Requerimientos funcionales(RF)

RF1 - Catalogo de Productos: El sistema debera mostrar en un catalogo los productos disponibles.

RF2- Agregar y quitar del carrito: El sistema debera permitirle al usuario agregar prodcutos y quitarlos del carrito.

RF3- Confirmacion: El Sistema debera permitr que el usuario confirme el pedido.

RF4- Estado del Pedido: El sistema debe generar el pedido con un estado inicial ir cambiando segun el avance.

RF5- Visualizacion: El personal del cafetin debe visualizar los pedidos realizados a través de una pantalla.


RF6- Actualizacion de Estado: El personal del cafetín debera actualizar el estado del pedido segun su avance.

Requerimientos no funcionales(RNF)

RNF1- Facil de Usar: La interfaz debe ser totalmente intuitiva y facil de usar para el personal del cafetín y los usuarios.

RNF2- Tiempo de respuesta: El sistema debera responder en intervalos de tiempo de 8 a 15 Segundos.

RNF3- Accesibilidad: El sistema debera ser totalmente accesible para los navegadores web.

RNF4-Informacion: El sistema debera mantener la informacion del pedido de forma consistente.

Reglas de negocio(RN)

RN1- Modificaciones: Un pedido no podra modificarse después de ser confirmado.

RN2 - Orden de estados: Todos los pedido deben pasar por cada estados en un orden lógico.

RN3 - Estado: Solo el personal del cafetín podra cambiar el estado del pedido y debera cambiarlo segun el estado del pedido.

RN4 - Producto: Despues de que el pedido se entregado, este no podra ser devuelto.
